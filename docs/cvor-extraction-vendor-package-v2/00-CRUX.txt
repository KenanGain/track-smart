WHAT WE WANT
------------
A REST API that ingests an Ontario MTO CVOR Carrier Safety Profile PDF and
returns a JSON document we can persist and render in our existing UI. One PDF
per request OR a bulk endpoint that accepts up to 50.

EVERY EXTRACTION MUST CONFORM TO:
   cvor-extraction-response.schema.json   (Draft-07)

THE 4 EXTRACTION SINKS  (matches the 4 colors on the highlighted PDFs)
----------------------------------------------------------------------
GREEN   Carrier Identity         One-time. CVOR #, name, address, status,
                                 dates, rating, fleet. Only on PDF page 1.
BLUE    Per-Pull Metric          One row per upload. R-Factor, OOS rates,
                                 event counts, kms, points. Pages 1-3.
PURPLE  Per-Inspection Event     CVIR-keyed events from PDF event log
                                 (pages 4-17). Inspection-typed only.
                                 Each carries violations[] and tickets[].
YELLOW  Optional / Audit         Order #, Search Date, US DOT, Mobile, Fax.

We do NOT extract: Conviction events, Collision events, Tow Operator section,
                   Travel Kilometric History (pages 18-19).

ACCEPTANCE
----------
1.  Output validates against cvor-extraction-response.schema.json.
2.  For PDF #1, output matches examples/response-single.json field-for-field
    (within tolerances in 04-validation.md).
3.  Bulk endpoint accepts 1-50 PDFs, returns per-file success or error.
4.  Single-PDF latency <= 30s synchronous.
5.  examples/expected/<pdf>.json is what your API should return for that PDF.
