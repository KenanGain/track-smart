> Adds vendor records and maintenance work orders for every carrier in `ACCOUNTS_DB` so the Inventory > Vendors page and the Asset Maintenance page work for all 30 carriers, not just Acme. No frontend changes — additions to data files only.

### Auto-generation rules per carrier

5 vendors per carrier with this category mix:

1. **Local repair shop** — `cat-repair-maintenance`. Name = `${city} Diesel Repair` (US) or `${city} Diesel Service` (CA). Address uses the carrier's city/state with a deterministic synthetic street + ZIP/postal code derived from the carrier's id hash.
2. **Local tire centre** — `cat-repair-maintenance`. Name = `${city} Tire & Wheel`. Same locality rule.
3. **Fuel-card vendor** — `cat-fuel-card`. Pulled from a 5-entry pool of national chains: Comdata · EFS · WEX Fleet · RTS Carrier · Esso Fleet (Imperial Oil for Canadian carriers).
4. **ELD provider** — `cat-eld-provider`. Pool: Samsara · Motive · Geotab · Verizon Connect.
5. **Dashcam / GPS** — `cat-dashcam` (Lytx, Netradyne) or `cat-gps-tracking` (Trimble Maps), rotating per carrier.

Vendor IDs are deterministic: `v-<short-account-id>-<role>` (e.g. `v-004-repair`, `v-007-fuelcard`). Same accountId → same set every reload.

## 4 · Work order coverage

| Carriers | Tasks | Orders |
| --- | --- | --- |
| acct-001 (Acme) | 23 hand-curated tasks across a1–a7 | 8 hand-curated orders |
| acct-002 → acct-030 | up to 20 tasks per carrier (1–2 per truck on the first 12 trucks) | 1 order per task-bundle (one order per asset) |

### Per-carrier generator rules

For each carrier:

1. Pull `assets = CARRIER_ASSETS[accountId]` and `vendorPool = repair-maintenance vendors for this carrier`.
2. Take the first 12 trucks (or fewer if the fleet is small). First 8 get 2 tasks each; the rest get 1 task each.
3. Each task picks a service type from a 6-template pool (oil, tires, brakes, annual, fifth-wheel grease, reefer) and rotates `status` across `upcoming / due / overdue / completed / in_progress`.
4. Tasks for the same asset get bundled into one `TaskOrder` whose `vendorId` is round-robined across the carrier's repair shops.
5. Order status is sampled deterministically: ~55% open, ~37% completed (with full invoice + cost breakdown), ~8% cancelled. Completed orders carry an invoice number, currency (CAD for Canadian carriers, USD otherwise), invoice date, parts/labour/tax/totalPaid, and remarks.
6. Every record uses real driver/asset references — `taskIds[]` chains a `MaintenanceTask`'s `assetId` to a real asset id in `CARRIER_ASSETS`, and `vendorId` to a real vendor id in `VENDORS`.

### Status / currency / odometer-unit rules

- **Currency** — `CAD` when `account.country === "CA"`, otherwise `USD`.
- **Odometer unit** — `km` for Canadian carriers, `miles` for US.
- **Engine-hours required** — only set when the task includes a reefer service.
- **Due dates / capture dates** — anchored to a 60–90 day window around today using a deterministic per-carrier RNG.

## 5 · Wiring

`maintenance.data.ts`:

```ts
const ACME_INITIAL_TASKS: MaintenanceTask[]   = [ /* hand-curated */ ];
const ACME_INITIAL_ORDERS: TaskOrder[]        = [ /* hand-curated */ ];

import { CARRIER_GENERATED_TASKS, CARRIER_GENERATED_ORDERS } from "./carrier-work-orders.data";

export const INITIAL_TASKS:  MaintenanceTask[] = [...ACME_INITIAL_TASKS,  ...CARRIER_GENERATED_TASKS];
export const INITIAL_ORDERS: TaskOrder[]       = [...ACME_INITIAL_ORDERS, ...CARRIER_GENERATED_ORDERS];
```

`inventory.data.ts`:

```ts
import { GENERATED_VENDORS } from "./carrier-vendor-seed.data";

const STATIC_VENDORS: Vendor[] = [ /* hand-curated for acct-001/002/003 */ ];

export const VENDORS: Vendor[] = [...STATIC_VENDORS, ...GENERATED_VENDORS];
```

The UI consumes `VENDORS` and `INITIAL_TASKS / INITIAL_ORDERS` exactly as before. The Asset Maintenance page already filters vendors by `accountId === account.id` (line ~663 of `AssetMaintenancePage.tsx`), so each carrier now sees its own 5 vendors in the dropdown.

## 7 · Acceptance

A carrier is "done" when:

1. `VENDORS.filter(v => v.accountId === accountId)` returns ≥ 3 vendors with the right `accountId`, valid email/phone, and one of the canonical category ids.
2. `INITIAL_ORDERS.filter(o => isOrderForCarrier(o, accountId))` returns ≥ 1 open and ≥ 1 completed order.
3. Every order's `vendorId` resolves through `getVendorById()` to a vendor with the same `accountId`.
4. Every order's `taskIds[]` resolves to tasks whose `assetId` is in `CARRIER_ASSETS[accountId]`.
5. `npx tsc -b` is clean.
